01

How it fits the plugin vision

PromptWise's thesis is orchestration over replacement: a conductor that sits above coding agents and rides open standards rather than rebuilding them. This feature is the most direct expression of that thesis yet — it is literally a conductor writing one score into each instrument's notation.

It isn't a new pillar. Your config_emitter.py already opens with: "Pillar 1 (agent-neutral portability): the same policy + packs + method renders into CLAUDE.md / AGENTS.md, .cursor/rules…". The plan below completes Pillar 1 by adding the three things a v1 emitter is missing: intent-awareness (so it knows what to write), agent-awareness (so it follows each tool's real rules), and a review loop (so the user stays in control).

  • Open standards as moat: AGENTS.md is now the shared base read by Codex, Cursor, Copilot, Gemini, Windsurf, Zed and more — so it becomes the canonical target, and per-tool files are emitted only as deltas where a tool adds capability plain markdown can't express.
  • Freemium-ready surface: detect + preview is free-tier; multi-repo sync, drift enforcement, and the compliance overlay are natural Team/Enterprise gates.
  • Regulated-industry credibility: the compliance gate and policy-as-code you already have become the non-negotiable sections injected into every agent file — auditable, consistent, verified.

03

Target architecture

Five stages, each a small composable unit. Stages 1–2 produce the content; 3–5 produce and reconcile the files.

# the pipeline (all stdlib + PyYAML; no network — matches existing constraints)

  user prompt
       │
  [1] CONTEXT   build_context_model()        # router + role_detector + repo scan
       │        → ProjectContextModel{intent, role, stack, domain, regulated?}
       │
  [2] BUNDLE    GovernanceBundle.from_context()  # + policy + compliance overlay + packs
       │
  [3] DETECT    detect_agents(repo_root)         # which agents are in play → targets
       │        → ["claude","cursor"]  (or explicit user list)
       │
  [4] COMPILE   ConfigEmitter.render(profile)    # profile-driven, per-agent format
       │        → managed-block content for each target file
       │
  [5] RECONCILE merge_managed(existing,new)      # preview · diff · apply, never clobber
       │        → lint(file, profile)            # token-tax / byte-cap / frontmatter
       ▼
  files written  +  diff shown to user  →  user edits freely

The arrows are pure function calls. Each stage is independently testable and independently shippable, which is what lets the rollout stay non-breaking.

05

The managed-block protocol

This is the mechanism that delivers "create the files automatically, then let the user change them." PromptWise owns a fenced region; everything outside it belongs to the user and is never touched on regeneration.

CLAUDE.md — after first syncgenerated + user-owned regions
<!-- promptwise:managed:start v=1 hash=8f3c… -->
# this project — agent guidance (generated by PromptWise)
Method: PromptWise governed agile method.
Governance policy (enforced via PromptWise):
- Block destructive ops without an audit record
- All AI-assisted changes require the quality gate
<!-- promptwise:managed:end -->

## Your notes (PromptWise will never edit below this line)
- Prefer pnpm; the legacy api/ folder is frozen.

Merge algorithm — three cases

State of file on diskAction
Does not existWrite managed block + an empty user section beneath it.
Exists with markersReplace only the bytes between :start and :end. Hash in the marker lets us skip writes when nothing changed (clean diffs, no churn).
Exists without markers (hand-authored)Never overwrite. Emit a diff and require an explicit adopt=true to wrap the existing content as the user section and insert the managed block above it.
core/config_emitter.pyadditive
MANAGED_START = "<!-- promptwise:managed:start v=1 hash={h} -->"
MANAGED_END   = "<!-- promptwise:managed:end -->"

def merge_managed(existing: str | None, new_block: str, *, adopt=False) -> str:
    """Return file content with ONLY the managed region replaced."""
    h = hashlib.sha256(new_block.encode()).hexdigest()[:8]
    block = f"{MANAGED_START.format(h=h)}\n{new_block}\n{MANAGED_END}"
    if existing is None:
        return block + "\n\n## Your notes (PromptWise will never edit below this line)\n"
    if MANAGED_START.split(" hash")[0] in existing:
        return re.sub(r"<!-- promptwise:managed:start.*?managed:end -->",
                      block, existing, flags=re.S)
    if not adopt:
        raise ConfigConflict("unmanaged file present; pass adopt=True to wrap it")
    return block + "\n\n## Your notes\n" + existing

For files that are wholly structural (Cursor .mdc with frontmatter), the same idea applies with an HTML-comment fence below the frontmatter, so user additions live after the managed body.

Agent profile registry — the dynamic mechanism

"If the user is on Claude, follow Claude's process; if Codex, follow Codex's" becomes a data lookup, not branching logic. Each agent is one declarative profile. Because formats change monthly, profiles are versioned data you can ship without a code release.

The profile schema

core/agent_profiles.pynew
@dataclass
class AgentProfile:
    key: str                      # "claude" | "cursor" | "codex" | "copilot" | "gemini"
    display_name: str
    targets: list[TargetFile]     # path + format + scope
    max_bytes: int | None = None   # Codex truncates ~32 KiB; warn before
    frontmatter: bool = False      # Cursor .mdc requires it
    supports_globs: bool = False   # Cursor/Copilot path-scoping
    activation_modes: list[str] = ()  # cursor: always|auto|agent|manual
    nested_hierarchy: bool = False  # Codex: root→cwd concat, nearest wins
    supports_imports: bool = False  # Claude @path includes
    always_on_token_budget: int = 0   # 0 = no cap; Cursor ~2000 soft cap
    commands_first: bool = True     # lead with setup/test/build

class ProfileRegistry:
    def get(self, key) -> AgentProfile: ...      # built-in default…
    def load_overrides(self, config_dir): ...     # …merged with YAML

Scope discipline
Because Codex, Cursor, Copilot and Gemini all read AGENTS.md, ship that as the canonical file first. Per-tool files are emitted only where the tool adds something AGENTS.md can't carry — Cursor's glob-scoped .mdc, Claude's @-imports. That keeps the v1 surface tiny and avoids five files drifting out of sync.

09

Zero-breaking-change guarantees

  • Additive modules. The 4 new files import nothing that imports them back; deleting them reverts to today's behavior.
  • Existing tool signatures frozen. All 65 current tools keep identical names, schemas, and outputs. sync_agent_config gains only optional args with defaults that reproduce current behavior.
  • Emitter back-compat shims. Old render(bundle, target) and sync(bundle, root, targets, write=) remain; new profile-driven paths are added alongside, with the old calls delegating to them.
  • Config is optional. No config/agent_profiles/ and no governance: block → built-in defaults run. Existing promptwise.yaml stays valid untouched.
  • Files on disk are safe. Managed blocks mean regeneration never touches user content; unmarked hand-authored files are refused unless explicitly adopted.
  • Same runtime constraints. Stdlib + PyYAML only, no network — consistent with policy.py / config_emitter.py today, so the sandbox and offline guarantees hold.
  • Detector is read-only. It only reads the filesystem; it never writes during detection.
