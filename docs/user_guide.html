PromptWise User Guide

Token-aware prompt routing and cost optimization · Claude Code MCP Plugin

PromptWise is a Claude Code MCP plugin that reduces LLM costs through smart model routing, prompt compression, prompt-cache planning, and usage tracking — all running locally with no external API calls.

What PromptWise Does

Every LLM call has three cost levers you can pull:

🎯
Right model — Haiku for extraction is 5× cheaper than Sonnet. Sonnet for multi-step agents is safer than Haiku.
✂️
Fewer tokens — Preamble phrases and filler words can account for 10–30% of a prompt. Strip them before sending.
💾
Cached tokens — Repeated system prompts and reference documents cost 10× less when cached (0.10/MTok vs 1.00/MTok for Haiku).

PromptWise provides 13 MCP tools that address each lever. Claude Code can call them automatically during your session, or you can invoke them explicitly via the /promptwise skill.

All local. PromptWise stores usage history in ~/.promptwise/history.db (SQLite). No data leaves your machine. Pricing data is in pricing.yaml inside the plugin directory.

Installation

  1. Add the local marketplace (one-time, already done if you see this guide): 
    ! /plugin add marketplace --directory C:\test\PromptWise promptwise-local
  2. Install the plugin: 
    ! /plugin install promptwise @ promptwise-local
  3. Reload plugins: 
    ! /reload-plugins
  4. Verify the MCP server started — you should see promptwise in the MCP list, and the promptwise:promptwise skill in the skills list.
Python requirement: PromptWise runs as python -m promptwise.server. Requires Python 3.11+ with mcp, pydantic, pyyaml, aiosqlite, and tiktoken installed. Run pip install -e . from the project root.

Quick Start

The fastest way to use PromptWise is to describe what you want. Claude Code will pick the right tool automatically when the promptwise:promptwise skill is active:

— "Which model should I use for this classification task?"
→ route_request called automatically

— "This prompt is too verbose, tighten it up"
→ rewrite_prompt called automatically

— "I'm building a chatbot that reuses the same system prompt 100×/day"
→ plan_cache called automatically

— "How much have I spent this session?"
→ get_session_stats called automatically

Or invoke directly via the slash menu:

! /promptwise:promptwise route_request

route_request

🎯 route_request
Picks the right model tier (fast / balanced / powerful) based on your task's intent, stakes, and optional budget constraints.
Use when: choosing between Haiku, Sonnet, and Opus; budget is tight; task stakes matter (production vs. scratch).
ParameterRequiredDescription
textreqDescribe the task or paste the prompt you're about to send
intentoptauto (default) or one of: extract, classify, summarize, question, code, analysis, agent_loop, research
stakesoptauto (default), low, medium, high. High stakes force Opus.
monthly_budget_usdoptYour monthly budget. If burn rate exceeds it, model is auto-downgraded.
days_elapsed_in_monthoptPaired with monthly_budget_usd to calculate projected monthly spend.

Example

Input:
{
  "text": "Extract all company names and their revenue from this earnings report",
  "intent": "extract",
  "stakes": "medium"
}

Output:
{
  "recommended_model": "claude-haiku-4-5-20251001",
  "reason": "Intent: extract, Stakes: medium, Tool chain: 0",
  "estimated_input_cost_usd": 0.000042,
  "context_window_pct": 0.2,
  "batch_recommended": true,
  "batch_recommendation_note": "Batch API available: 50% cost reduction...",
  "alternatives": [...]
}
Budget guard: if you pass monthly_budget_usd: 50 and days_elapsed_in_month: 25, PromptWise projects your spend to month-end and downgrades the tier if you're over 110% of budget.

rewrite_prompt

✂️ rewrite_prompt
Strips preamble phrases ("could you please", "I was wondering if") and filler words ("just", "basically", "really"), then optionally prepends a role-specific frame.
Use when: prompt has conversational padding; want role-specific framing; want to reduce input tokens before a repeated call.
ParameterRequiredDescription
textreqThe original prompt to rewrite
roleoptgeneral (default), developer, analyst, writer, manager, researcher, designer
modeloptModel for token counting (default: claude-sonnet-4-6)

Role Prefixes

RolePrefix added
developer"From a software engineering perspective, "
analyst"With supporting data and structured analysis, "
manager"For strategic decision-making, "
researcher"With methodical analysis and evidence-based reasoning, "
writer"With clear prose and narrative flow, "
designer"From a user experience and visual design perspective, "

Example

Input text:
"Hey Claude, could you please just basically help me debug this Python function?
I was wondering if you could really look at the logic here."

Output:
{
  "rewritten": "From a software engineering perspective, help me debug this Python function? look at the logic here.",
  "saving_pct": 28.6,
  "warning": null
}

optimize_context

📦 optimize_context
Trims a large context block (document, codebase, conversation history) to fit a token budget by dropping low-value sentences.
Use when: pasting a large document; context exceeds 80% of the model's window; wanting to reduce cost on a repeated reference.
ParameterRequiredDescription
contextreqThe text to compress
token_budgetoptTarget max tokens (default: 2000, min: 100)
modeloptModel for token counting
Output:
{
  "optimized": "... compressed text ...",
  "saving_pct": 61.3,
  "chunks_dropped": 14
}
Note: optimize_context uses extractive sentence scoring — it keeps the highest-scored sentences up to the token budget. For documents where every sentence matters (legal, medical), review the output before use.

plan_cache

💾 plan_cache
Analyzes a message list and recommends where to place Anthropic prompt-cache breakpoints (cache_control markers). Shows projected savings vs. no caching.
Use when: building a chatbot, RAG app, or agentic loop that reuses the same system prompt or reference document across many calls.
ParameterRequiredDescription
messagesreqArray of {role, content, label?}. Labels: tool_definitions, reference, conversation_history
expected_reuse_countoptHow many times this message set will be sent (default: 2)
modeloptModel for token counting and pricing

Cache TTL rules

Message typeTTLWhy
role: system1hAlways stable across calls
label: tool_definitions1hTool schemas rarely change mid-session
label: reference ≥ 1000 tok1hBreak-even at 2 reuses; profitable with more
label: conversation_history > 2000 tok5mHistory changes per turn; short TTL appropriate
Example output:
{
  "breakpoints": [
    { "message_index": 0, "ttl": "1h", "rationale": "system prompt (stable across calls)" },
    { "message_index": 1, "ttl": "1h", "rationale": "large reference (4200 tokens, break-even at 2 reuses)" }
  ],
  "savings_pct": 72.4
}

batch_prompts

📋 batch_prompts
Merges 2–5 related small tasks into a single numbered prompt, saving the per-call overhead tokens (role framing, context reload).
Use when: you have 2–5 similar classification, extraction, or summarization tasks to run back-to-back.
ParameterRequiredDescription
tasksreqArray of 2–5 task strings
roleoptRole framing for the combined prompt (default: general)
modeloptModel for token counting
Input tasks:
["Summarize this article in 3 bullets", "Extract the 5 key dates mentioned", "Rate the sentiment: positive/neutral/negative"]

Output:
{
  "batched_prompt": "Complete all 3 tasks:\n1. Summarize this article in 3 bullets\n2. Extract the 5 key dates mentioned\n3. Rate the sentiment...",
  "saving_pct": 18.4
}

summarize_thread

🧵 summarize_thread
Compresses a long conversation into a compact summary and generates a reset prompt you can paste at the start of a new thread.
Use when: a conversation is approaching context limits; wrapping up a long debugging session; handing off to a fresh thread.
ParameterRequiredDescription
conversationreqThe conversation text to summarize
max_tokensoptTarget summary length in tokens (default: 500, max: 2000)
modeloptModel for token counting
Output:
{
  "summary": "We debugged the auth middleware. Root cause: JWT expiry check used strict less-than...",
  "reset_prompt": "Continue from: [summary]. Next step: write the unit test.",
  "saving_pct": 84.2
}

compare_providers

⚖️ compare_providers
Estimates the cost of a given prompt across all configured providers and tiers, sorted cheapest first. Includes context-window fit check.
Use when: evaluating whether to switch providers for a workload; responding to "is this cheaper on GPT?"
ParameterRequiredDescription
textreqThe prompt to cost-compare
modeloptModel used for token counting (default: claude-sonnet-4-6)
Output (sorted by total_cost_usd):
{ "comparisons": [
  { "provider": "gemini", "tier": "fast", "model": "gemini-2.0-flash", "total_cost_usd": 0.0000048 },
  { "provider": "openai", "tier": "fast", "model": "gpt-4o-mini", "total_cost_usd": 0.0000216 },
  { "provider": "claude", "tier": "fast", "model": "claude-haiku-4-5-20251001", "total_cost_usd": 0.000072 },
  ...
]}
OpenAI and Gemini pricing in pricing.yaml is approximate. Verify against official pricing pages before making production decisions.

get_session_stats

📊 get_session_stats
Returns cumulative usage statistics: total cost, savings, cache hit rate, calls by tool, and cost by model.
ParameterRequiredDescription
sinceoptISO 8601 timestamp to filter results (e.g. 2026-05-01T00:00:00Z)
Output:
{
  "total_calls": 47,
  "total_cost_usd": 0.004821,
  "total_savings_usd": 0.001923,
  "avg_saving_pct": 28.5,
  "cache_hit_rate": 0.64,
  "calls_by_tool": { "route_request": 12, "rewrite_prompt": 18, ... },
  "cost_by_model": { "claude-haiku-4-5": 0.0012, ... },
  "stale_pricing_warning": null
}
If pricing.yaml was last verified more than 90 days ago, a stale_pricing_warning is included in the response reminding you to update rates.

🔔 ping_session
Creates or updates a session record. Returns a session_id you can pass to subsequent calls to track idle time.

🔄 reload_config
Hot-reloads pricing.yaml, providers.yaml, and roles.yaml without restarting the MCP server. Call after editing any config file.

No parameters. Returns {"reloaded": true} on success.

Configuration Files

PromptWise reads three YAML files from its plugin directory at startup. Edit them to customize pricing, add providers, or change roles.

pricing.yaml

Defines per-model rates (input/output/cache/batch) in USD per million tokens, plus context window size and max output.

Example model entry:
claude-sonnet-4-6:
  display_name: "Sonnet 4.6"
  provider: claude
  tier: balanced
  context_window: 1000000
  max_output: 128000
  rates:
    input_per_mtok: 3.00
    output_per_mtok: 15.00
    cache_write_5m_per_mtok: 3.75
    cache_write_1h_per_mtok: 6.00
    cache_hit_per_mtok: 0.30
    batch_input_per_mtok: 1.50
    batch_output_per_mtok: 7.50
Update last_verified in pricing.yaml each time you verify rates. PromptWise will warn you if it's more than 90 days old.

providers.yaml

Maps provider names to tier→model assignments. The tiers block must reference model IDs that exist in pricing.yaml.

roles.yaml

Defines role prefixes (prepended by rewrite_prompt) and the filler words / preamble phrases to strip.

promptwise.yaml (optional)

Override session timeout defaults:

timeout:
  idle_threshold_minutes: 30
  warn_threshold_minutes: 20
  auto_action: prompt_user   # or summarize_and_exit

Workflow: RAG / Agentic App

1️⃣
Call plan_cache with your message template (system prompt + tool definitions + few-shot examples). Set expected_reuse_count to your daily call volume.
2️⃣
Add cache_control: {type: "ephemeral"} breakpoints at the recommended message_index positions in your actual API calls.
3️⃣
Call route_request with the user query to pick the right tier. For retrieval, intent: "question" usually routes to balanced.
4️⃣
At session end, call get_session_stats to verify cache hit rate and actual savings.

Workflow: Reducing Daily Dev Spend

1️⃣
Each morning: call route_request before any long task. Ask "is this Opus-level work?" Most code tasks are Sonnet.
2️⃣
Before sending a drafted prompt: call rewrite_prompt. Even 10% token reduction compounds significantly over many calls.
3️⃣
For large documents pasted as context: call optimize_context with a budget that fits the task (2000 tokens for simple Q&A, 8000 for analysis).
4️⃣
When you have 3+ small tasks: call batch_prompts instead of sending each separately.

Workflow: FinOps Reporting

1️⃣
Call export_stats with format: "csv" and since: "2026-05-01T00:00:00Z" for monthly data.
2️⃣
Import into Excel/Sheets. The project and team columns support cost allocation (populated via stats.record() in custom integrations).
3️⃣
Call clear_history with older_than_days: 90 to enforce retention policy.
